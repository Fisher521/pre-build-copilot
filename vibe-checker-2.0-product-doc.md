# Vibe Checker 2.0 产品优化文档

## 一、产品定位

### 目标用户：Vibe Coder

**用户画像：**
- 不一定懂技术，主要靠 AI + 低代码工具拼产品
- 想法多，但不确定哪个值得做
- 希望快速验证，不想花太多时间
- 一个人做，不涉及团队协作
- 追求"能用就行"，不追求完美

**他们不是：**
- 正经创业者（不需要商业计划书）
- 专业开发者（不需要复杂架构）
- 有线下资源的人（做不了 O2O）

### 核心价值

帮 Vibe Coder 回答三个问题：
1. **能不能做？** → 技术可行性评估
2. **值不值得做？** → 市场机会分析
3. **怎么开始？** → 最快上手路径

### 设计原则

| 原则 | 说明 |
|-----|-----|
| 用人话 | 不用 MVP、技术栈、冷启动这些术语，或用了立刻解释 |
| 渐进式清晰 | 用户不需要想清楚再输入，AI 帮他理清思路 |
| 问卷即启发 | 每个问题都在帮用户思考，不只是收集信息 |
| 给选择不给答案 | 方案对比让用户自己判断，不是"唯一正确答案" |
| 能劝退就劝退 | 不适合 vibe coder 做的项目，直接说 |
| 学习内嵌 | 用完不只得到报告，还顺便学到东西 |

## 二、用户流程

### 整体流程

```
自由输入 → AI 解析 → 确认修改 → 动态追问 → 过程总结 → 生成报告
```

### Step 1：自由输入

**界面：**
```
💡 告诉我你想做什么

┌─────────────────────────────────────────┐
│ 随便说说你的想法，不用想得太清楚...        │
│                                         │
│                                         │
└─────────────────────────────────────────┘

比如：
[帮人快速生成周报的工具]  [独居老人订菜小程序]  [宠物健康追踪 App]
```

**设计要点：**
- 输入框足够大，支持多行
- 提供示例降低输入门槛
- 支持语音输入
- 不要求格式，随便说

---

### Step 2：AI 解析 + 确认修改

**AI 输出结构化理解：**
```
✨ 我理解了你的想法

┌─────────────────────────────────────────┐
│ 📌 项目名称                              │
│ 宠物健康追踪工具                          │ [🎤]
│                                         │
│ 🎯 核心功能                              │
│ 记录宠物的健康数据（体重、饮食、就医记录）   │ [🎤]
│                                         │
│ 👤 目标用户                              │
│ 养宠物的人，想更好地管理宠物健康           │ [🎤]
│                                         │
│ 💡 解决的问题                            │
│ 宠物健康信息分散，不方便查看和管理          │ [🎤]
└─────────────────────────────────────────┘

哪里说得不对？可以直接改，或者语音补充

[← 重新描述]                    [确认继续 →]
```

**设计要点：**
- AI 主动填充，用户只需确认或修改
- 每个字段可单独编辑
- 支持语音补充
- 如果信息太少，AI 会标注"待补充"而非瞎猜

---

### Step 3：动态追问

**问题选择逻辑：**
```
AI 识别项目类型 → 从问题库抽取相关问题 → 生成 3-4 个问题
```

**问题结构（每题包含）：**
- 问题本身
- 点拨（为什么问这个）
- 选项
- 选完后的即时反馈

---

#### 状态题（必选 1 个）

根据项目类型选择不同问法：

**通用版：**
```
你之前最接近的经历是？

💡 这会帮我调整建议的详细程度

○ 从没用 AI 做过东西，今天想试试
○ 跟着教程做过，但自己想做的没做出来
○ 做过一些小东西，想做个正经点的
○ 做过好几个了，想看看这个想法靠不靠谱
```

| 选项 | 即时反馈 |
|-----|---------|
| 从没做过 | 那我会给你最详细的步骤，每一步都带说明，别担心 |
| 跟着教程做过 | 我会重点标出容易卡住的地方，帮你避坑 |
| 做过一些小东西 | 基础的我就不啰嗦了，重点讲这个项目特殊的地方 |
| 做过好几个了 | 直接讲关键决策点，不浪费你时间 |

---

**工具类项目：**
```
你现在是怎么解决这个问题的？

💡 了解你现在的痛苦程度，能帮我判断这个想法的价值

○ 忍着，每次都很烦
○ 用别的工具凑合
○ 手动搞，费时间但能解决
○ 没解决，想看看能不能做一个
```

---

**交易/服务类项目：**
```
你能接触到第一批用户吗？

💡 很多产品做完了才发现不知道给谁用，早点想清楚能省很多弯路

○ 就是给自己或家人用
○ 有一小群人我能直接联系到
○ 得想办法找
○ 没想过这个
```

---

#### 项目相关题库

**产品形态：**
```
你希望做成什么样子？

💡 不同形式的难度差很多，选对了能省很多事

○ 网页，打开就能用
  → ✅ 最简单的形式，几小时就能上线
  
○ 微信小程序
  → ⚠️ 需要注册和审核，比网页多一周流程
  
○ 浏览器插件
  → 📋 中等难度，要学一点插件开发的规则
  
○ 手机 App
  → ⚠️ 最复杂，建议先用网页验证想法
  
○ 不确定，你帮我建议
  → 我会根据你的想法推荐最省事的形式
```

---

**竞品认知：**
```
有没有你觉得类似的产品？

💡 了解竞品能帮你少走弯路，也能发现机会

○ 有，但我觉得可以做得更好
  → 💡 我会帮你分析它们的弱点在哪
  
○ 有，但我想做个不一样的方向
  → ✅ 差异化是对的，我会帮你想想怎么定位
  
○ 没见过
  → 可能是蓝海，也可能是伪需求，我帮你分析
  
○ 没调研过
  → 我帮你查一下，看看市面上有什么
```

---

**变现意愿：**
```
你希望它能赚钱吗？

💡 这会影响我给你的建议方向

○ 不用，做着玩 / 自己用
  → ✅ 那专注做得好用就行
  
○ 能赚点零花钱就行
  → 我会建议一些简单的变现方式
  
○ 希望能当副业
  → 会帮你想想定价和获客
  
○ 没想过
  → 先做出来，后面再说也行
```

---

**AI 相关项目：**
```
AI 生成的结果不完美，你能接受吗？

💡 AI 输出有不确定性，想清楚这个会影响产品怎么设计

○ 可以，给个初稿自己改
  → ✅ 这个定位最现实，也最容易做
  
○ 希望尽量准
  → 需要在提示词和流程上多下功夫
  
○ 必须很准
  → ⚠️ 这很难做到，建议降低预期或加人工审核
```

---

**内容类项目：**
```
内容从哪来？

💡 内容来源决定了你的工作量和能不能持续

○ 我自己写/整理
  → 想清楚能坚持多久，内容产品最怕断更
  
○ AI 帮我生成
  → ✅ 省力，但要注意质量把控
  
○ 抓取/聚合现有的
  → ⚠️ 注意版权问题和数据源稳定性
  
○ 还没想好
  → 建议先确定这个再动手，不然容易做一半卡住
```

---

**数据敏感度（工具类/AI类）：**
```
用户的数据敏感吗？

💡 这会影响你的技术选择和用户信任

○ 不敏感，随便存
  → ✅ 那技术选择很灵活，可以用最简单的方案
  
○ 有点，不想让别人看到
  → 建议用正规的云服务，别自己存
  
○ 很敏感（比如健康、财务）
  → 要认真考虑数据安全，可能需要额外投入
  
○ 没想过
  → 我会在报告里帮你分析需不需要特别处理
```

### 3.7 可能踩的坑

```
⚠️ 这个项目容易踩的坑

1. 想做的功能太多，一直做不完

   📖 经验：先砍到只剩一个核心功能
   比如你想做"宠物健康管理"，先只做"记录体重"这一个功能。
   能用 > 完美。上线了再慢慢加。

2. 花太多时间在界面细节上

   📖 经验：丑但能用 > 好看但没做完
   用户要的是解决问题，不是看设计。
   v0 生成的默认样式就够用了，别一直调。

3. 做完不知道发到哪里

   📖 经验：做之前就想好给谁用
   你的目标用户在哪里？即刻、小红书、还是微信群？
   做的时候心里有这个画面，做出来才知道往哪发。
```

## 四、劝退机制

### 什么情况下劝退

```
识别到以下特征时，触发劝退：

1. 涉及线下运营
   - 需要对接商家/供应商
   - 需要物流/配送
   - 需要线下服务

2. 技术复杂度超出 vibe coder 能力
   - 需要自己训练 AI 模型
   - 需要复杂的后端架构
   - 需要处理大量并发

3. 强监管领域
   - 金融/支付（需要牌照）
   - 医疗健康（需要资质）
   - 教育（需要备案）

4. 冷启动极难
   - 双边市场（买家卖家都要有）
   - 社区产品（需要大量用户才有价值）
   - 网络效应依赖
```

### 劝退话术

```
⚠️ 这个想法可能不太适合一个人快速做出来

我理解你想做：帮独居老人订菜的小程序

但这个项目有几个硬伤：

1. 需要搞定供菜的商家
   这不是写代码能解决的，要一家一家去谈

2. 涉及资金流转
   用户付钱、你收钱、再给商家，支付和合规很复杂

3. 需要持续运营
   不是做完就能放着的，每天都有订单要处理

💡 如果你真的想做这个方向：

可以换一个"轻"一点的切入角度：
• 做一个"老年人友好的买菜指南"——推荐附近哪些菜店送货、怎么联系
• 做一个"帮子女给父母订菜的提醒工具"——只做提醒，不做交易
• 做一个"菜品价格比较"——纯信息聚合，不涉及交易

这些不需要搞定供应商，技术上简单很多，一个人就能做。

[换一个想法重新评估]    [看看类似的简单项目]
```

根据项目类型，推荐最省事的工具组合：

#### 场景 A：纯静态网页（展示、简单交互）

```
v0.dev / bolt.new  →  Vercel / Cloudflare Pages
     生成代码              部署上线

国产替代：扣子空间 / Trae + Zeabur
```

#### 场景 B：需要用户登录 + 存数据

```
v0.dev / bolt.new  →  Supabase / MemFire  →  Vercel
     生成代码           用户 + 数据库          部署

国产替代：Trae + MemFire Cloud + Zeabur
```

#### 场景 C：AI 应用（调用大模型）

```
Cursor / Trae  →  DeepSeek API / 通义千问  →  Vercel / Zeabur
   写代码              AI 能力                 部署

最省钱：DeepSeek API（国产最便宜）
```

#### 场景 D：微信小程序

```
扣子空间（最快）
或
uni-app + 微信云开发（更灵活）
```

#### 场景 E：浏览器插件

```
Plasmo + Cursor  →  Chrome Web Store
   开发框架             发布
```

### 九、国内访问建议

对于不方便科学上网的用户：

| 需求 | 国际工具 | 国产替代 |
|-----|---------|---------|
| AI 代码生成 | v0 / bolt.new | 扣子空间 / Trae / 豆包MarsCode |
| AI 编辑器 | Cursor | Trae / 通义灵码 |
| 部署托管 | Vercel | Zeabur / Cloudflare Pages |
| 后端数据库 | Supabase | MemFire Cloud / LeanCloud |
| AI API | OpenAI | DeepSeek / 通义千问 / 智谱 |

💡 推荐组合（国内友好）：
```
Trae（编辑器）+ 扣子空间（快速原型）+ MemFire（数据库）+ Zeabur（部署）+ DeepSeek（AI）
```
